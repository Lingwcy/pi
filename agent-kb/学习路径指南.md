# 🎓 小白专属：Agent 架构学习路径指南

面对包含 20 篇深度架构文档的知识库，如果你感觉无从下手，请不要慌张。

本指南是为你量身定制的“课程表”。我们将复杂的 Agent 系统拆解为四个循序渐进的阶段。请严格**按照顺序**阅读以下推荐的文章，每读完一篇，不妨停下来思考一下“如果让我来写，我会怎么处理这个问题？”

---

## 🟢 第一阶段：建立直觉 (建立宏观认知)
*在这个阶段，我们不看任何代码，只搞清楚系统里有哪些角色，以及它们要解决什么现实问题。*

1. **先看主页** ➡️ [[README]]
   - **重点看什么**：看看那张宏观的 Mermaid 架构图。只要记住系统分为三层：最外层的“UI”，中间做管理的“Harness（编排器）”，和最底层干苦力的“Loop（执行引擎）”。
2. **认识两位主角** ➡️ [[核心概念/AgentHarness]] 和 [[核心概念/AgentLoop]]
   - **重点看什么**：弄清楚它们俩的分工。为什么 `AgentLoop` 不能直接接收 UI 的点击事件？理解 Harness 就像是汽车的方向盘和外壳，而 Loop 是发动机。

## 🟡 第二阶段：理解“保护机制” (系统为什么不会崩)
*当你让 AI 去写代码时，它可能要花 1 分钟。在这 1 分钟里，用户如果狂点界面，系统为什么不崩溃？这是本架构最精华的部分。*

3. **读懂状态隔离** ➡️ [[核心概念/状态模型]]
   - **重点看什么**：这是全书**最重要**的一篇文章！必须理解什么是“活配置 (Live Config)”，什么是“回合快照 (Turn Snapshot)”。理解系统是如何把用户的修改“暂存”起来的。
4. **掌握红绿灯** ➡️ [[核心概念/生命周期]]
   - **重点看什么**：看那张状态机图（`idle` -> `turn`）。理解当阶段变成 `turn` (忙碌) 时，系统是如何拒绝那些会引起灾难的操作的。仔细阅读“单次对话的微观旅程”，它把 Agent 思考的几秒钟像慢动作一样拆解了。

## 🟠 第三阶段：扩展能力的魔法 (如何给 Agent 装插件)
*现在你懂得了系统如何安全地运转。接下来我们要学习：如果我想让 Agent 每次发消息前都自动附带当前的时间，该怎么做？*

5. **认识事件系统** ➡️ [[核心概念/钩子与事件]]
   - **重点看什么**：忘掉你以前学过的普通的事件监听（EventEmitter）。重点理解这里的三种插件模式：“转换链”、“补丁累加”和“短路拦截”。
6. **看看代码怎么写** ➡️ [[参考资料/钩子设计]]
   - **重点看什么**：这是你接触的第一篇涉及深层设计的文章。看看 TypeScript 是如何用 `symbol` (幻影类型) 强制规定插件不能乱返回数据的。看懂那几段伪代码（Reducer），你会大呼过瘾。

## 🔴 第四阶段：代码实战与底层黑科技 (成为架构师)
*恭喜你来到深水区！现在你已经具备了阅读源码的理论基础。*

7. **源码导读** ➡️ [[核心组件/AgentLoop函数]] 和 [[核心组件/AgentHarness类]]
   - **重点看什么**：去看看我们之前学的“状态锁”、“快照”和“钩子”，在代码里是如何用 `while` 循环和 `Promise.all` 表达出来的。特别是 Loop 函数中的“工具并发执行与有序回收”算法。
8. **时空穿梭的秘密** ➡️ [[底层基建/对话树与持久化引擎]]
   - **重点看什么**：打破常识：聊天记录不能存成数组！看看系统是如何用 JSONL 文件和 `parentId` 建立一棵对话树的。这解释了为什么你可以让 Agent 无限次回退重试而数据不乱。
9. **跨端运行的秘密** ➡️ [[底层基建/跨平台环境沙箱]]
   - **重点看什么**：理解“依赖倒置”原则。明白为什么在核心代码里绝对不能写 `import fs from 'fs'`。

---

> [!success] 结业挑战
> 如果你能流利地向同事解释以下三个问题，那么恭喜你，你已经完全吃透了这套架构：
> 1. 为什么在 `AgentHarness` 中，不仅有消息队列（Steer），还有一个 `PendingWrites`（挂起写入）队列？
> 2. 大模型一次性要求调用 3 个工具，系统在并行执行它们时，是如何保证最终写入数据库的顺序不乱的？
> 3. 系统如果崩溃断电，重启后是如何从 JSONL 文件中精确恢复到崩溃前那个分支的状态的？

祝你学习愉快！
