# 📖 第一篇：核心概念 (Core Concepts)

欢迎来到核心概念章节。在深入研究复杂的 TypeScript 接口和异步算法之前，建立正确的**系统心理模型 (Mental Model)** 是至关重要的。

本章节将向你展示 `packages/agent` 架构背后最深刻的设计哲学：它是如何通过多层隔离、严格的状态机和巧妙的事件流，来驯服大语言模型（LLM）这个充满不确定性的“黑盒”的。

## 🎯 本章学习目标

通过阅读本章，你将掌握：
1. **架构分层思想**：为什么系统必须拆分为主理业务的 `AgentHarness` 和主理执行的 `AgentLoop`？
2. **并发防御机制**：在长时间的 AI 对话中，如何保证用户的实时修改不会导致内存指针异常或时间线错乱？
3. **极客级别的扩展性**：如何设计一套允许第三方插件“篡改”数据但又绝对类型安全的 Hook 系统？

### 1. 架构的双子塔
我们首先要认识系统中的两位主角：
- **[[AgentHarness]] (编排器)**：系统的“大管家”。它处理着所有来自人类的复杂请求（点击、打断、修改配置），它将所有的混乱挡在门外，只把最纯净的任务交给底层。
- **[[AgentLoop]] (执行循环)**：系统的“V8 引擎”。它是高度专注的，完全与平台无关。它唯一的使命就是与 LLM 搏斗，执行那些复杂的 JSON 工具调用。

### 2. 防御的护城河
理解了角色后，我们要理解它们是如何在恶劣环境下生存的：
- **[[状态模型]]**：如果 Agent 正在运行，用户突然删掉了一个工具，会发生什么？这篇文章将带你理解系统是如何通过“快照隔离”和“延迟写入”来化解危机的。
- **[[生命周期]]**：任何伟大的系统都有其呼吸的节奏。这篇文章详细拆解了一次简单的 `prompt()` 请求，在底层经历了怎样的 7 步微观旅程才得以安全落盘。

### 3. 扩展的桥梁
当你理解了核心系统的坚固之后，最后我们要学习如何安全地扩展它：
- **[[钩子与事件]]**：传统的事件监听器太弱了。我们将展示如何利用 TypeScript 构建一条带有防呆设计的“责任链”流水线，让插件可以大胆地拦截和修改系统行为。
